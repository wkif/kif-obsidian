# 🧠 Kif's Obsidian Vault

> 个人知识管理系统 | Personal Knowledge Management System

## 📚 简介

这是我的个人 Obsidian 知识库，用于管理技术学习、项目开发、灵感想法和内容创作。采用混合式组织结构，结合 PARA 方法论和十进制分类系统。

## 🗂️ 目录结构

### 🎯 核心导航
- **00-Dashboard** - 控制中心，展示当前重点和目标

### 📅 时间管理
- **01-Daily** - 每日记录
- **02-Weekly** - 每周复盘

### 💻 技术领域
- **10-Tech** - 技术总览和通用技术笔记
- **11-Frontend** - 前端技术（React、Vue、CSS 等）
- **12-Backend** - 后端技术（Node.js、Django、数据库等）
- **13-AI** - 人工智能、机器学习、LLM 相关

### 📁 项目管理
- **20-Projects** - 正在进行的项目

### 💡 创意想法
- **30-Ideas** - 想法汇总
- **31-Product-Ideas** - 产品创意
- **32-Side-Projects** - 个人 Side Project

### 📚 知识沉淀
- **40-Knowledge** - 知识库总览
- **41-Concepts** - 核心概念和原理

- **50-Content** - 内容规划
- **51-Blog-Drafts** - 博客草稿

- **90-Archive** - 已完成或不再活跃的内容

## 🚀 快速开始

1. **从 Dashboard 开始**：打开 `00-Dashboard/README.md` 查看当前重点
2. **每日记录**：在 `01-Daily` 创建今日笔记
3. **学习笔记**：根据主题在对应技术分类下创建笔记
4. **项目管理**：在 `20-Projects` 跟踪项目进展

## 📝 使用原则

### 笔记组织
- 使用双向链接连接相关笔记
- 每个笔记专注一个主题
- 善用标签进行横向分类
- 定期整理和归档

### 命名规范
- 文件名使用英文，避免特殊字符
- 使用 PascalCase 或 kebab-case
- 包含日期的文件：`YYYY-MM-DD-title.md`

### 标签系统
- `#todo` - 待办事项
- `#in-progress` - 进行中
- `#done` - 已完成
- `#reference` - 参考资料
- `#question` - 待解决问题

## 🎯 工作流

### 每日流程
1. 查看 Dashboard 了解当前重点
2. 创建/更新 Daily Note
3. 处理待办事项
4. 记录学习内容和灵感

### 每周流程
1. 周日或周一创建 Weekly Note
2. 回顾上周完成内容
3. 规划本周目标
4. 整理和归档笔记

### 项目流程
1. 在 `20-Projects` 创建项目笔记
2. 链接相关技术笔记
3. 跟踪进度和问题
4. 完成后移至 Archive
