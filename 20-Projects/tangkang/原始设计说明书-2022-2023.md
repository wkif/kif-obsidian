## 二、小程序设计（2022-2023版）

### 2.1 小程序入口方式
1. **微信搜索**: "糖康有道"
2. **扫码进入**: 专用小程序码
3. **朋友分享**: 社交分享功能

### 2.2 首页模块设计

#### 2.2.1 首页布局（原始设计）
```
A区域: 公告轮播区域（已移除）
B区域: 资讯搜索
C区域: 置顶推荐资讯轮播
D区域: 最近血糖记录
E区域: 资讯列表
```

#### 2.2.2 快捷操作区（原始功能）
| 序号 | 名称       | 功能说明                 |
|------|------------|--------------------------|
| 1    | 积分       | 用户积分查看和管理       |
| 2    | 饮食记录   | 饮食热量记录和计算       |
| 3    | 运动打卡   | 运动记录和热量消耗       |
| 4    | 食品数据库 | 食物营养信息查询         |
| 5    | 制定目标   | 个性化健康目标设定       |

### 2.3 记录模块设计

#### 2.3.1 血糖值日记录
- **数据可视化**: 折线统计图展示近一周/月血糖趋势
- **详情查看**: 点击数据点查看具体记录信息
- **新增记录**: 支持空腹、餐后等不同时段记录

#### 2.3.2 不定期记录
- **糖化血红蛋白**: 3个月平均血糖水平
- **尿微量白蛋白**: 肾脏功能筛查
- **足背静脉搏动**: 血管功能评估

### 2.4 "我的"模块设计

#### 2.4.1 用户信息管理
- **登录授权**: 微信一键登录
- **健康信息**: 近7天健康数据概览
- **个人信息**: 基本健康信息管理

#### 2.4.2 积分系统
**积分来源**（原始设计）:
1. 血糖值达到目标值
2. 每日运动量达标
3. 绑定监护人
4. 每日打卡
5. 资讯浏览时长
6. 邀请他人注册
7. 分享达标计划

#### 2.4.3 亲友关爱功能
- **关联家人**: 支持用户名/手机号搜索
- **数据共享**: 查看关联用户7天内健康数据
- **权限管理**: 接受/拒绝关联请求

#### 2.4.4 其他功能
- **地址管理**: 多地址管理，设置默认地址
- **语音播报**: 针对老龄用户的语音功能
- **关于我们**: 项目信息和更新日志
- **用户协议**: 服务协议展示
- **联系客服**: 微信客服系统集成

## 七、设计演变与对比

### 7.1 从原始设计到当前系统（2026年）

| 功能模块 | 原始设计（2022-2023） | 当前系统（2026年） |

| **医生端** | 无 | ✅ 完整医生管理系统 |
| **组织架构** | 无 | ✅ 跨医院医生组织管理 |
| **健康食谱** | 基础食品数据库 | ✅ AI智能食谱推荐 |
| **云服务** | 阿里云 | ✅ 百度云迁移完成 |

#### 技术升级
| 技术方面 | 原始设计 | 当前系统 |
|----------|----------|----------|
| **通信协议** | 未明确 | ✅ SSE替代WebSocket |
| **性能优化** | 未提及 | ✅ 分包加载，性能提升 |
| **安全认证** | 基础登录 | ✅ 多用户认证系统重构 |
| **部署方式** | 手动部署 | ✅ Docker容器化+Jenkins |

#### 用户体验改进
| 体验方面 | 原始设计 | 当前系统 |
|----------|----------|----------|
| **界面统一** | 基础界面 | ✅ 统一UI风格 |

| **加载速度** | 未优化 | ✅ 首次加载<1.5s |

### 7.2 架构演变
1. **服务器迁移**: 阿里云 → 百度云（2026年完成）
2. **通信升级**: 基础HTTP → SSE实时通信
3. **部署优化**: 手动部署 → Docker自动化
4. **安全增强**: 基础认证 → 多用户安全体系
