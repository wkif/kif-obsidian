## 一、项目概述

### 1.1 项目定位
唐康医疗健康平台是一个面向糖尿病患者、医疗工作者及健康管理人群的综合性数字健康管理平台，提供从个人健康管理到医患协同的全方位服务。

### 1.2 核心价值
- **患者端**: 便捷的健康数据记录、智能分析、医患互动、个性化指导
- **医生端**: 患者管理、远程指导、健康干预、团队协作
- **平台端**: 数据整合、智能提醒、健康知识传播、电商服务

### 1.3 发展历程
- **2022年8月**: 项目启动，基础糖尿病管理小程序
- **2023年3月**: 功能完善，后台管理系统上线
- **2026年1-4月**: 技术架构全面升级，功能扩展
- **当前**: 成熟的医疗健康管理平台

## 二、系统架构

### 2.1 技术架构概览

#### 前端架构
```
微信小程序（主入口）
├── 主包：核心功能（首页、记录、我的）
├── 分包1：商城、聊天、医生端
├── 分包2：资讯内容
├── 分包3：健康食谱
└── 技术栈：原生小程序 + Vant Weapp + ECharts
```

#### 后端架构
```
Django REST Framework（核心API）
├── 用户服务：认证、权限、个人信息
├── 健康数据服务：记录、分析、统计
├── 即时通讯服务：SSE实时通信
├── 医生管理服务：组织架构、患者管理
├── 内容服务：资讯、食谱、知识库
└── 电商服务：商品、订单、支付
```

#### 部署架构
- **云服务**: 百度云（已完成阿里云→百度云迁移）
- **容器化**: Docker + Docker Compose
- **自动化部署**: Jenkins流水线
- **数据库**: PostgreSQL + Redis缓存

### 2.2 性能优化
- **分包加载**: 主包+分包策略，首次加载<1.5s
- **图片优化**: 懒加载 + WebP格式
- **数据缓存**: 合理使用本地存储和Redis
- **请求合并**: 减少不必要的网络请求

### 2.3 安全架构
- **传输安全**: HTTPS全程加密
- **数据安全**: 敏感数据加密存储
- **访问控制**: RBAC权限管理系统
- **隐私保护**: 隐私模式、数据脱敏
- **合规性**: 符合医疗数据管理法规

├── 消息接收和处理

├── SSE连接管理
